## 二、高价值优化点

### 2.1 首轮查询 - 参数解析扩展

| 用例表达 | 当前 | 建议 |
|----------|------|------|
| **EV-091** 「离地铁0.5公里」 | 可能未解析 | 0.5公里→500m，补充 `(\d+\.?\d*)\s*公里` |
| **EV-184** 「地铁距离调整到300米以内」 | 多轮可能未更新 | 明确「调整到X米」覆盖上一轮 max_subway_dist |
| **EV-032** 「希望附近有健身房」首轮 | 仅 district+price+bedrooms | 健身房可作首轮 tag 意向，或首轮宽松查再筛选 |

## 三、按优先级排序的改动建议

### P0（易实现、高收益）

1. **补充 _SUBWAY_STATIONS**：中关村站、中关村
2. **0.5公里/公里单位解析**：`(\d+\.?\d*)\s*公里` → max_subway_dist
3. **租房短路扩展**：「就第一套吧」「我们租」「就定第一套，我们租」
4. **房山城关**：输入预处理去掉「房山房山」重复

### P1（中等收益）

5. **拎包入住** → 精装（或 家具齐全 若 API 有对应 tag）
6. **网红餐饮** → 近餐饮
7. **加油站** → 近加油站（需确认 API）
8. **晨跑、户外跑步** → 近公园

### P2（需架构支持）

9. **双区域**：丰台或朝阳 → 多 district 查询合并
10. **预算继承与调整**：多轮「压一半」「提高到」从上一轮改写 max_price
11. **室友数量**：希望有室友且只有1个 → 合租 + 无直接 tag，可作软过滤
